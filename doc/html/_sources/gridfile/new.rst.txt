gridfile.new
============
Create a new, empty .grid file

----

Syntax
------

.. raw:: html

    <pre>

.. rst-class:: syntax

| :ref:`obj = gridfile.new(filename, metadata) <gridfile.new.syntax1>`
| :ref:`obj = gridfile.new(filename, metadata, overwrite) <gridfile.new.syntax2>`

.. raw:: html

    </pre>

----

Description
-----------

.. raw:: html

    <pre>

.. _gridfile.new.syntax1:

.. rst-class:: syntax

| :ref:`obj <gridfile.new.output.obj>` = gridfile.new(:ref:`filename <gridfile.new.input.filename>`, :ref:`metadata <gridfile.new.input.metadata>`)

.. raw:: html

    </pre>

Initializes a new (empty) .grid file. The new file manages an N-dimensional grid that spans the provided metadata. Adds a ".grid" extension to the filename if it does not already have one.


.. raw:: html

    <pre>

.. _gridfile.new.syntax2:

.. rst-class:: syntax

| :ref:`obj <gridfile.new.output.obj>` = gridfile.new(:ref:`filename <gridfile.new.input.filename>`, :ref:`metadata <gridfile.new.input.metadata>`, :ref:`overwrite <gridfile.new.input.overwrite>`)

.. raw:: html

    </pre>

Specify whether the method may overwrite pre-existing .grid files. By default, the method will not overwrite files.

filename
++++++++

.. raw:: html

    <section class="accordion"><input type="checkbox" name="collapse" id="input1" checked="checked"><label for="input1"><strong>filename</strong></label><div class="content">

| *string* *scalar*
| The name of the new .grid file. May be a filename, relative path, or absolute path. If not an absolute path, saves the new file relative to the current directory.

    </div></section>

metadata
++++++++

.. raw:: html

    <section class="accordion"><input type="checkbox" name="collapse" id="input2" checked="checked"><label for="input2"><strong>metadata</strong></label><div class="content">

| *gridMetadata* *object*
| Metadata for the gridded dataset. Metadata rows must be unique for each dimension. If the metadata has a set dimension order, the order will be ignored.

    </div></section>
